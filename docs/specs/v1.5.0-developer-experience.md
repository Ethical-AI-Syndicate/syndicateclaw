# v1.5.0 — Developer Experience

**Sprint Duration:** 3 weeks
**Goal:** Make SyndicateClaw accessible to developers through SDK, visual tools, and extensibility

---

## 1. Problem Statement

SyndicateClaw's API is comprehensive but has a high barrier to entry: workflows are defined in raw JSON, there's no Python SDK, and there's no way to visualize workflow graphs. Developers must read API docs, manually construct requests, and debug via HTTP responses. This limits adoption and slows iteration.

## 2. Goals

- **Python SDK**: Typed, Pythonic client for all API endpoints
- **Visual workflow builder**: Web UI to create and edit workflows
- **Plugin system**: Extensible architecture for custom node handlers and tools

## 3. Non-Goals

- Mobile SDK (future)
- JavaScript/TypeScript SDK (future)
- Marketplace for plugins (future)
- Workflow templates (future)

## 4. Python SDK

### 4.1 Package

```
pip install syndicateclaw-sdk
```

### 4.2 Client Design

```python
from syndicateclaw_sdk import SyndicateClaw

# Initialize
client = SyndicateClaw(
    base_url="https://syndicateclaw.example.com",
    api_key="sk-..."
)

# Or with token
client = SyndicateClaw.from_token(
    base_url="https://syndicateclaw.example.com",
    token="eyJ..."
)
```

### 4.3 API Methods

```python
# Workflows
workflow = client.workflows.create(
    name="my-workflow",
    description="A simple workflow",
    nodes=[
        {"id": "start", "type": "START", "handler": "start"},
        {"id": "end", "type": "END", "handler": "end"},
    ],
    edges=[{"from_node_id": "start", "to_node_id": "end"}]
)

workflow = client.workflows.get("wf-abc123")
workflows = client.workflows.list(status="active")
client.workflows.delete("wf-abc123")

# Runs
run = client.workflows.runs.create("wf-abc123", input_state={"key": "value"})
run = client.workflows.runs.get("wf-abc123", "run-xyz789")
client.workflows.runs.pause("wf-abc123", "run-xyz789")
client.workflows.runs.resume("wf-abc123", "run-xyz789")

# Streaming
async for event in client.workflows.runs.stream("wf-abc123", "run-xyz789"):
    print(event.type, event.data)

# Memory
client.memory.write("my-namespace", "key1", {"value": "data"})
record = client.memory.read("my-namespace", "key1")
results = client.memory.search("my-namespace", query="data")

# Tools
client.tools.execute("http_request", {"url": "https://api.example.com"})

# Agents
agent = client.agents.register(name="my-agent", capabilities=["research"])
client.agents.heartbeat("my-agent")
msg = client.messages.send(recipient="other-agent", content={"action": "do_something"})
reply = client.messages.reply(msg.id, content={"result": "done"})

# Approvals
pending = client.approvals.list(status="PENDING")
client.approvals.approve("approval-abc123")
client.approvals.reject("approval-abc123", reason="Insufficient evidence")

# Organizations
org = client.organizations.create(name="acme-corp")
client.organizations.add_member("org-abc123", actor="user@example.com", role="MEMBER")
```

### 4.4 Workflow Builder

A fluent API for constructing workflows programmatically:

```python
from syndicateclaw_sdk.builder import WorkflowBuilder

wf = (
    WorkflowBuilder("invoice-approval")
    .description("Automated invoice approval with human review")
    .start("init")
    .action("validate", handler="validate_invoice")
    .decision("high_amount", condition="state.invoice_amount > 5000")
        .when(True).approval("manual_review", assigned_to=["finance@company.com"])
        .when(False).action("auto_approve", handler="approve_invoice")
    .checkpoint("after_approval")
    .end("complete")
    .build()
)

client.workflows.create_from_builder(wf)
```

### 4.5 Local Development Mode

For testing workflows without a running server:

```python
from syndicateclaw_sdk.local import LocalRuntime

# In-memory execution (no Postgres needed)
runtime = LocalRuntime()
result = runtime.execute(wf, input_state={"invoice_amount": 3000})
print(result.state)  # {"invoice_amount": 3000, "auto_approved": true}
```

### 4.6 Async Support

All methods have async variants:

```python
# Sync
result = client.workflows.create(...)

# Async
result = await client.workflows.async_create(...)
```

### 4.7 Error Handling

```python
from syndicateclaw_sdk.exceptions import (
    WorkflowNotFoundError,
    ToolDeniedError,
    ApprovalRequiredError,
    RateLimitError,
    AuthenticationError,
)

try:
    client.tools.execute("http_request", {"url": "https://blocked.internal"})
except ToolDeniedError as e:
    print(f"Tool denied: {e.reason}")
except RateLimitError as e:
    print(f"Rate limited. Retry after {e.retry_after}s")
```

## 5. Visual Workflow Builder

### 5.1 Architecture

A React-based web UI that:

1. Renders workflow graphs visually (using react-flow)
2. Supports drag-and-drop node creation
3. Allows edge editing and condition configuration
4. Imports/export JSON workflow definitions
5. Communicates with the SyndicateClaw API

### 5.2 Components

| Component | Description |
|-----------|-------------|
| `WorkflowCanvas` | Main graph canvas (react-flow) |
| `NodePalette` | Drag-and-drop node types (START, END, ACTION, DECISION, APPROVAL, CHECKPOINT) |
| `NodeConfig` | Side panel for configuring selected node |
| `EdgeConfig` | Side panel for configuring selected edge |
| `StatePreview` | Live preview of workflow state |
| `RunControls` | Start/pause/resume/cancel buttons |

### 5.3 Embedding

The builder is served as a standalone React app, but can be embedded via iframe:

```html
<iframe
  src="https://syndicateclaw.example.com/builder?workflow_id=wf-abc123&token=..."
  width="100%"
  height="800px"
></iframe>
```

### 5.4 API Integration

The builder communicates with the existing API:

- `GET /api/v1/workflows/{id}` — Load workflow definition
- `PUT /api/v1/workflows/{id}` — Save workflow definition
- `POST /api/v1/workflows/{id}/runs` — Start a run
- `GET /api/v1/runs/{id}/stream` — Stream events for visualization

### 5.5 Code View

Toggle between visual and code views:

- **Visual**: React-flow graph with node/edge configuration panels
- **Code**: Raw JSON editor (Monaco editor for syntax highlighting)
- **Diff**: Compare current edit with saved version

### 5.6 Embedded Builder Endpoint

```
GET /builder/{workflow_id}
GET /builder/new
```

Serves the builder HTML page with the workflow loaded.

## 6. Plugin System

### 6.1 Plugin Interface

```python
from syndicateclaw.plugins import Plugin, PluginContext

class MyPlugin(Plugin):
    name = "my-plugin"
    version = "1.0.0"

    async def on_workflow_start(self, ctx: PluginContext):
        """Called when a workflow execution begins."""
        pass

    async def on_node_execute(self, ctx: PluginContext, node_id: str, result: Any):
        """Called after each node executes."""
        pass

    async def on_workflow_end(self, ctx: PluginContext, status: str):
        """Called when a workflow execution completes."""
        pass

    async def on_error(self, ctx: PluginContext, error: Exception):
        """Called on workflow error."""
        pass
```

### 6.2 Plugin Registration

```python
# At startup
from syndicateclaw.plugins import PluginRegistry

registry = PluginRegistry()
registry.register(MyPlugin())
registry.register(AuditTrailPlugin())
registry.register(CostTrackingPlugin())
```

### 6.3 Plugin Sandbox

Plugins run in a sandboxed context:

- **Read-only state**: Plugins can read workflow state but not modify it directly.
- **Audit trail**: Plugin actions are audited separately.
- **Timeout**: Plugin execution is bounded (default: 5 seconds per hook).
- **Error isolation**: Plugin errors don't crash the workflow.

### 6.4 Built-in Plugins

| Plugin | Description |
|--------|-------------|
| `AuditTrailPlugin` | Enhanced audit logging with custom fields |
| `CostTrackingPlugin` | Track LLM token costs per workflow |
| `RateLimitPlugin` | Custom rate limiting rules |
| `WebhookPlugin` | Send webhooks on workflow events |
| `MetricsPlugin` | Custom Prometheus metrics |

### 6.5 Plugin Discovery

Plugins can be loaded from:

1. **Built-in**: `syndicateclaw.plugins.builtin`
2. **Package**: `pip install syndicateclaw-plugin-*` (auto-discovered via entry points)
3. **File**: Load from a Python file path

```yaml
# plugins.yaml
plugins:
  - syndicateclaw.plugins.builtin.AuditTrailPlugin
  - syndicateclaw.plugins.builtin.MetricsPlugin
  - path: /path/to/custom_plugin.py
    class: MyCustomPlugin
```

## 12. Risks

| Risk | Mitigation |
|------|------------|
| SDK API drift | Auto-generate SDK from OpenAPI spec |
| Builder complexity | Start with read-only view; editing is additive |
| Plugin security | Sandboxed execution; timeout enforcement; audit trail |
| SDK packaging | Use `hatchling` for builds; `twine` for PyPI |

## 13. Rollback

- SDK: remove PyPI package; existing API clients unaffected
- Builder: disable endpoint (return 404)
- Plugins: disable plugin loading; core functionality unaffected
